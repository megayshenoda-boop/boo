### Arena Flow

1. Client requests match info: 0x0CF4 (ARENA_MATCH_INFO_REQUEST_NEW)
2. Server sends opponents: 0x05EE (ARENA_MATCH_INFO_RETURN)
3. Client can refresh opponents: 0x0CF5 (ARENA_CHANGE_MATCH_REQUEST_NEW)
4. Client challenges: 0x05F1 (ARENA_CHALLENGE_MATCH_REQUEST)
5. Server returns result: 0x05F2 (ARENA_CHALLENGE_MATCH_RETURN)
6. Ranking updates: 0x05F4 (SYNC_ARENA_INFO)

### Arena Hero Queue

- 0x05E6 SYNC: Server sends current hero lineup
- 0x05E7 REQUEST: Client changes hero lineup for arena
- 0x05E8 RETURN: Server confirms change

### Arena Automation

- Buy attempts: 0x05EC (fire-and-forget? check for RETURN)
- Restore attempts: 0x05EB (fire-and-forget)
- Auto-challenge: loop 0x0CF4 -> 0x05F1 -> wait 0x05F2 -> repeat
- Note: 0x0CF4 and 0x0CF5 use NEW encrypted format (CMsgCodec)

## 7. BOT AUTOMATION OPPORTUNITIES
--------------------------------------------------

### Auto-Arena Bot Flow:
```
1. Send 0x0CF4 (ARENA_MATCH_INFO_REQUEST_NEW) - get opponents
2. Parse 0x05EE response - pick weakest opponent
3. Send 0x05F1 (ARENA_CHALLENGE_MATCH_REQUEST) - start fight
4. Wait for 0x05F2 (ARENA_CHALLENGE_MATCH_RETURN) - result
5. If attempts exhausted: 0x05EB or 0x05EC to get more
6. Repeat until daily limit
```

### Auto-Expedition Bot Flow:
```
1. Send 0x02B2 (EXPEDITION_INFO_REQUEST) - get available stages
2. Parse 0x02B3 response
3. Send 0x02B6 (EXPEDITION_BATTLE_REQUEST) for each stage
4. Repeat until all stages cleared
```

### Reconnaissance:
```
1. 0x029E (RANK_INFO_REQUEST) - get top players
2. 0x0176 (BATTLE_DETAIL_REPORT_REQUEST) - read battle reports
3. Use data to choose weakest targets for attack
```

## 8. VULNERABILITY NOTES
--------------------------------------------------

- May allow unlimited arena attempts if server doesn't track cooldown
- Test: Send 0x05EB repeatedly, check if attempts increase

### [MEDIUM] Arena Match Refresh Spam

- Keep refreshing until you find weak opponents

- Unlikely to be exploitable but worth testing
