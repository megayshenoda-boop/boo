## Potential Vulnerabilities & Exploits
============================================================

### Shop Exploits

1. **Camel Shop Refresh** (0x0392): No RETURN pair - could refresh without cost check
2. **Lucky Shop Scratch** (0x09A8): Fire-and-forget - may allow rapid scratching
3. **Royal Shop Refresh** (0x0798): Forced refresh could cycle inventory
4. **Reward Point Shop**: Buy items with points - check if points deducted server-side only

### Reward Duplication Risks

1. **Fire-and-forget rewards**: No return = no client confirmation needed
   - Send reward opcode multiple times rapidly
   - Server may not have rate limiting on all endpoints
2. **Event reward timing**: Query info, collect reward, re-query
   - Race condition between info sync and reward collection
3. **Batch rewards**: Some systems have individual + batch reward opcodes
   - Collecting both may yield double rewards

### Chat Exploits

1. **Chat Gift** (0x027C-0x027E): Gift sending via chat - check cost validation
2. **Chat Bubble Buy** (0x0278): Purchase without gems check?
3. **Chat Block/Unblock**: Rapid block/unblock for notification spam
4. **Group Chat**: Create unlimited groups, add members without consent

### Event Manipulation

1. **Cycle/Operation/Rush Action**: All follow INFO->REWARD pattern
   - Reward without meeting requirements if server doesn't validate
2. **Kingdom Action Extra** (0x0B64): Extra opcode with no RETURN
3. **War Lord Extra** (0x138E): Same pattern - extra action without response
4. **Mobilization** (0x07B2-0x07CA): Large system - 25 opcodes
   - Multiple reward endpoints to test
   - Task completion may not be validated
